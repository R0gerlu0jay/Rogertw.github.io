Git漏洞復現，Clone一個項目就能RCE?!

Roger Lv1
  2024-11-13 19:30:48 2024-11-13 19:30:48 Created   2024-11-14 11:54:20 2024-11-14 11:54:20 Updated  
  • CVE 
  • | 漏洞復現 
 

摘要

  • CVE編號：CVE-2024-32002
  • 受影響系統：Windows,Mac
  • Git影響版本:2.45.0, 2.44.0, <=2.43.3, <=2.42.1, 2.41.0, <=2.40.1, <=2.39.3
  • 漏洞嚴重性：高（CVSS評分：9.1）

攻擊原理

Git hook

Git Hook是Git的機制，通常在Git Clone一個項目或是將本地的倉庫初始化git init後，”本地”的git客戶端都會生成一個.git目錄，這些Hook文件存在.git/hooks/目錄當中(附圖⤵️)

Git Hook允許開發者在特定的Git操作發生時自動執行腳本或命令，其中又有分成Client-Side Hooks和Server-Side Hooks


Client-Side Hooks

  • pre-commit:在執行git commit前觸發，可用於檢查程式格式、運行測試等等……。
  • prepare-commit-msg:在提交訊息編輯完成後觸發，用於檢查格式
  • pre-push：在執行git push操作之前觸發，可以進行測試、檢查或確認操作。
  • post-commit：在成功執行git commit後觸發，用於執行一些後續的操作，比如發送通知。

Server-Side Hooks

  • pre-receive：在伺服器接收到推送時觸發，用於檢查推送的內容是否符合要求。
  • post-receive：在推送完成後觸發，可以用來自動部署代碼或通知相關人員。
  • update：在推送到特定分支時觸發，通常用於更細粒度的檢查，例如限制某些分支的推送。

.git目錄的內容是由git程序自主維護，不會被Git項目的內容影響，當上傳項目時.git目錄也不會被傳到服務器上，所以正常情況從服務器clone一個項目只是把項目內容拉到本地，不會有惡意的Hook腳本(除非你手動將腳本拉到裡面)。

這是的攻擊手法是利用Git子模塊的特性將攻擊腳本寫入.git目錄裡

Git 子模塊(Submodule)

Git子模塊是Git的一個特性，可以將一個Git項目作為另一個Git項目的子目錄，可以在一個項目中包含其他項目，例如一個開源項目中包含另一個開源項目，主項目下面會有一個.gitmodules文件，裡面記錄了子模塊的相關信息(如下圖⤵️)

path: 子模塊的路徑 / url: 子模塊的git項目地址

如果我們在進行git clone時加上--recursive參數，Git會自動初始化子模塊，並將子模塊的內容遞歸的拉到本地，這時候子模塊的.git目錄也會被拉到本地，並放入對應的文件目錄位址，不僅主項目會有一個.git目錄，子模塊也會有一個.git目錄，不過這個.git目錄是空的，只有一個文件.git，裡面對應了這個子模塊真正的.git目錄的位址(如下圖⤵️)

sha1collisiondetection是子項目

Git 符號連結 (symbolic link)

Git符號連結是文件系統的特性，允許一個文件或目錄指向另一個文件或目錄，本身不包含文件內容，而是包含指向目標文件或目錄的路徑，當訪問符號連結時，系統會自動轉向目標文件或目錄，就像快捷方式一樣。

漏洞成因

前面有提到Hook腳本本身與項目內容無關，除非是手動放置腳本到hooks目錄中，否則項目中的內容是不會跑到.git目錄裡。


攻擊者製作了一個Git項目，在這個項目中又包含了一個子模塊，
當使用--recursive參數clone這個項目時，就會遞歸的去拉取項目放到對應的位置，下圖為該項目的.gitmodules文件

圖片中可以看到，他指示Git把url中的項目放到A/modules/x目錄中，然後在這個項目下，有一個名叫a的符號連結，並且指向.git目錄(如下圖⤵️)

由於Windows和Mac的文件和目錄名稱是大小寫不敏感，導致在放到A/modules/x實際上就是放到.git/modules/x，前面提過這個目錄是歸子模塊所屬的.git目錄，還按照原本的.git目錄結構，裡面有一個hooks目錄，這時候就會把攻擊者的Hook腳本放到這個目錄中，當你執行下個git clone時，就會執行這個腳本，這樣就可以達到RCE的目的，以下展示整個目錄的結構

漏洞復現

Mac Air M3 , git version 2.39.3

只要能彈出計算機就代表RCE成功了

想自己試試看的可以參考這個repo

結論

  • 平時還是要做好更新的習慣，及時更新Git版本
  • 不要隨便clone不明來源的項目，尤其是有子模塊的項目
